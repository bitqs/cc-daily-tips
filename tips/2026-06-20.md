# 2026-06-20 | settings.json 权限白名单：彻底告别「是否允许」弹窗

## 技巧

在 `.claude/settings.json` 里配置 `permissions.allow` / `permissions.deny`，把高频只读操作预先放行、把危险命令永久拦截——再也不用每次手动点「Allow」。

## 怎么做

**1. 在项目根目录创建或编辑 `.claude/settings.json`**

```bash
mkdir -p .claude
```

**2. 写入权限规则**

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Bash(git status)",
      "Bash(git log *)",
      "Bash(git diff *)",
      "Bash(git add *)",
      "Bash(npm run *)",
      "Bash(npx tsc *)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(rm -rf *)",
      "Bash(DROP *)",
      "Bash(truncate *)"
    ]
  }
}
```

- `allow` 里的工具调用**直接通过**，不弹确认框
- `deny` 里的模式**永远拦截**，Claude 会收到拒绝提示，不会误操作
- 支持通配符 `*`，`Bash(git *)` 表示所有 git 子命令

**3. 也可以配到全局（对所有项目生效）**

```bash
# 文件位置：~/.claude/settings.json
```

全局配置适合放「我从不需要确认的」基础读操作；项目级配置放项目特有的命令。

**4. 用内置 skill 自动生成白名单（懒人方案）**

```
/fewer-permission-prompts
```

这个 skill 会扫描你的历史 transcript，把高频只读命令自动写入项目 `settings.json`，省去手动枚举。

## 为什么

Claude Code 默认对每一类新操作弹窗确认，保护性强但很打断心流——尤其是 `git log`、`npm run build` 这种反复出现的无害命令。

配好白名单之后：
- **Read / Grep / Glob 类操作**：全程无感，Claude 直接查文件
- **`git push --force`、`rm -rf`**：永久拦截，彻底防止手滑
- 白名单存在 git 里，团队共享，新同事 clone 即用

痛点是：边写代码边被弹窗打断，注意力碎片化。这个配置一次搞定，永久省心。
