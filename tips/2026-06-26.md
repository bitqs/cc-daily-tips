# 2026-06-26 | SessionStart Hook：每次开会话，让 Claude 先"读一遍项目现状"

## 技巧

在 `.claude/settings.json` 里配置 `SessionStart` hook，每次新建会话时自动跑一段 shell 命令，把输出注入给 Claude——它开口说第一句话之前就已经知道你的 git 状态、最近提交和待处理变更。

## 怎么做

在项目根目录的 `.claude/settings.json` 加入以下配置：

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "echo '=== 当前分支 ===' && git branch --show-current && echo '=== 最近 5 次提交 ===' && git log --oneline -5 && echo '=== 工作区状态 ===' && git status --short"
          }
        ]
      }
    ]
  }
}
```

保存后重启 Claude Code，下次 `claude` 启动时会自动执行这段命令，输出作为上下文注入会话。

你也可以换成更复杂的脚本，比如跑测试摘要、检查 TODO 数量、或打印 `.env` 里缺少的变量：

```json
{
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/session-init.sh"
          }
        ]
      }
    ]
  }
}
```

## 为什么

正常情况下，打开 Claude Code 要先跟它解释一遍「我在哪个分支、改了什么、现在想干什么」。会话一多，这段背景交代又重复又烦。

SessionStart hook 把这件事变成零成本：Claude 启动的一瞬间就自动感知项目状态，你直接下指令就行——省掉的不是几秒钟，是每次"重新建立上下文"的心智负担。
