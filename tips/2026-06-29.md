# 2026-06-29 | `/init`：让 Claude Code 给自己写使用说明书

## 技巧

在项目根目录启动 Claude Code 后输入 `/init`，它会自动扫描整个代码库，生成一份结构化的 `CLAUDE.md`——技术栈、目录结构、常用命令、注意事项一并写好，不用你从零手敲。

## 怎么做

```bash
# 1. 进入项目根目录
cd your-project

# 2. 启动 Claude Code
claude

# 3. 在对话框里直接输入
/init
```

Claude Code 会：
1. 扫描项目文件、`package.json` / `Makefile` / CI 配置等
2. 识别技术栈、测试命令、构建流程
3. 在根目录生成（或更新）`CLAUDE.md` 初稿
4. 列出它填写的各节内容，供你确认或微调

生成后只需补充你的个人偏好（代码风格、禁止操作、特殊注意点），一份可用的项目说明书就完成了。

## 为什么

手动写 `CLAUDE.md` 费时且容易漏细节——尤其是接手陌生项目或老项目补文档时。`/init` 让 Claude 做"第一遍阅读"，自动整理出项目骨架，你只负责审阅和补充主观偏好。相当于让 AI 先替你读一遍项目，再替你写说明书。

## 发帖草稿

很多人都知道 CLAUDE.md 能给 Claude Code 提供项目背景，但不知道 Claude Code 可以给自己写使用说明书——

进项目目录，启动 Claude Code，输入 `/init`，它会扫完整个代码库，自动生成 CLAUDE.md：技术栈、构建命令、目录结构全都有。你只需要加几行个人偏好，省掉九成手动配置时间。

接手新项目或者老项目补文档，这个命令能帮你快速破冰。你的 CLAUDE.md 是自己写的还是让 Claude 写的？
