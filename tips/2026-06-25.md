# 全局 CLAUDE.md：个人偏好写一次，所有项目永久生效

## 技巧

在 `~/.claude/CLAUDE.md` 写下你的个人编码偏好，Claude Code 在任何项目里都会自动加载它——再也不用每次重新交代"你是谁、你喜欢什么风格"。

## 怎么做

创建或编辑全局记忆文件：

```bash
# 创建全局记忆文件
mkdir -p ~/.claude
vi ~/.claude/CLAUDE.md
```

写入你的个人偏好，例如：

```markdown
# 我的全局偏好

## 身份背景
我是一名独立开发者，主要用 TypeScript / Python，偶尔写 Go。

## 代码风格
- 变量命名用 camelCase（TS）/ snake_case（Python）
- 函数保持单一职责，不超过 40 行
- 注释只写"为什么"，不写"是什么"
- 优先使用标准库，避免引入不必要的依赖

## 回复偏好
- 直接给解决方案，不要先问一堆问题
- 代码示例必须可直接运行
- 中文回答

## 禁止事项
- 不要自动帮我 git commit
- 不要在未经确认时删除文件
```

**加载优先级**：全局 `~/.claude/CLAUDE.md` 会先被读取，项目级 `.claude/CLAUDE.md`（或根目录 `CLAUDE.md`）后加载并可覆盖全局设置。

验证是否生效：

```bash
# 在任意项目里启动 Claude Code，输入 /status 可看到加载的 memory 文件列表
claude
> /status
```

## 为什么

每次在新项目里用 Claude Code，你都要花几轮对话让它"认识你"——你的技术栈、代码风格、禁忌操作……这些重复的背景交代既浪费上下文 token，又浪费你的时间。

全局 CLAUDE.md 解决的是这个问题：它是你的「AI 工作搭档简历」，写一次，所有项目共享。项目级 CLAUDE.md 负责项目特定知识（架构、API 约定等），全局文件负责你这个人的不变偏好——两层分工清晰，互不干扰。

对于同时维护多个项目的开发者，这个文件能让你在任何一个新 repo 里，打开 Claude Code 就像和老搭档继续工作。

## 发帖草稿

`~/.claude/CLAUDE.md`，一个「全局记忆文件」。
