# 2026-06-21 | /compact：上下文快满时别 /clear，用这个续命

## 技巧

长会话上下文快撑满时，用 `/compact` 代替 `/clear`——它会把对话历史压缩成摘要继续当前会话，所有背景照样保留，不用从头再来。

## 怎么做

**1. 基础用法：一键压缩整段对话**

在 Claude Code 里直接输入：

```
/compact
```

Claude 会把整段对话历史提炼成一份结构化摘要，然后以这份摘要作为新的上下文继续工作。Token 占用大幅下降，但你们讨论过的决策、改过的文件、约定的规范都还在。

**2. 带焦点指令：告诉它该留什么**

```
/compact 只保留和支付模块相关的上下文，忽略 UI 部分的讨论
```

```
/compact focus on the auth refactor, drop all the database migration discussion
```

附上指令后，Claude 在压缩时会优先保留你关心的内容，裁掉已经完成或无关的部分——比无脑压缩更精准。

**3. 典型使用时机**

- 重构一个大模块聊了 1 小时，中途收到上下文警告
- 同一会话里先改 API 层、再改 UI 层，UI 部分已完成，现在要做测试
- 长时间 debug，前面很多来回已经是废话

```
# 示例：已完成 API 改造，现在聚焦测试
/compact 保留 API 路由改动的最终方案，丢弃中间的调试过程，现在开始写集成测试
```

**4. 和 `/clear` 的区别**

| | `/clear` | `/compact` |
|---|---|---|
| 对话历史 | 全部清空 | 压缩为摘要保留 |
| 已有上下文 | 丢失 | 保留关键信息 |
| Token 占用 | 归零 | 大幅降低 |
| 使用场景 | 完全换话题 | 同一任务继续做 |

## 为什么

Claude Code 的上下文有长度上限。很多人遇到上下文警告时，第一反应是 `/clear` 重开——但这意味着之前讲好的架构决策、代码规范、调试结论全部归零，要重新交代一遍才能继续。

`/compact` 解决的正是这个问题：

- **不断链**：同一个会话里继续，Claude 还记得你们一起改了哪些文件、做了哪些决定
- **可控压缩**：加上焦点指令，只压缩掉真正无用的历史，保留当前任务最需要的信息
- **省重复沟通**：对于半天的大型重构任务，中途压缩一次比重开会话少交代大量背景

## 发帖草稿

Claude Code 会话跑太久上下文快撑爆了？别直接 `/clear`——那会让它忘掉你们讨论过的所有东西。

试试 `/compact`，加上你想保留的方向：

```
/compact 保留支付模块的改动方案，丢掉前面的 UI 讨论
```

它会把整段历史压缩成摘要继续工作，关键决策不丢，Token 占用骤降——同一个会话接着干，不用从头交代背景。
