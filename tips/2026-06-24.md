# 2026-06-24 | Stop Hook：Claude Code 跑完任务，自动给你推系统通知

## 技巧

配置 `Stop` hook，让 Claude Code 在每次结束一轮回答时自动执行一条 shell 命令——比如弹出系统通知。跑长任务时可以放心去干别的，完事了电脑自己叫你。

## 怎么做

在 `.claude/settings.json`（项目级）或 `~/.claude/settings.json`（全局）里加：

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude Code 完成啦 ✅\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

**Linux 用户**把 command 换成：

```bash
notify-send "Claude Code" "任务完成啦 ✅"
```

保存后立即生效，无需重启。下次让 Claude Code 跑一个耗时任务，离开去喝杯水——回来时通知已经在等你了。

## 为什么

Claude Code 跑复杂任务（写大量代码、跑测试、分析日志）往往需要好几分钟。大多数人只能盯着终端干等，或者切去干别的但不知道啥时候完。

`Stop` hook 是 Claude Code 内置的「任务结束」事件，触发时机精确——不是轮询、不是猜，而是 Claude 真正停下来的那一刻。配合系统通知，你的注意力可以完全解放。

同理，这个 hook 还可以用来：
- 播放提示音（`afplay /System/Library/Sounds/Glass.aiff`）
- 发一条 Slack/企微消息（配合 curl）
- 记录完成时间到日志文件

## 发帖草稿

让 Claude Code 跑长任务时你一直盯着屏幕等？

配一个 Stop Hook，它跑完就自动弹系统通知——

```json
"Stop": [{ "hooks": [{ "type": "command",
  "command": "osascript -e 'display notification \"完成啦\" with title \"Claude Code\"'" }] }]
```

加进 `.claude/settings.json`，保存即生效。以后让 AI 干活，你去冲咖啡，回来通知等着你。
