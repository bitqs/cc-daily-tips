# 2026-06-22 | PreToolUse Hook：在 Claude Code 动手之前，先过你这关

## 技巧

在 `.claude/settings.json` 里配置 `PreToolUse` hook，Claude Code 每次调用工具**之前**先跑你的脚本——脚本退出码非 0 就直接拦截，工具根本不会执行。

## 怎么做

**1. 编写拦截脚本** `.claude/hooks/pre-tool-guard.sh`：

```bash
#!/usr/bin/env bash
# 从 stdin 读取 Claude 即将执行的工具信息（JSON）
input=$(cat)
tool=$(echo "$input" | jq -r '.tool_name')
cmd=$(echo "$input"  | jq -r '.tool_input.command // ""')

# 拦截所有包含 "rm -rf" 的 Bash 命令
if [[ "$tool" == "Bash" && "$cmd" == *"rm -rf"* ]]; then
  echo "⛔ 已拦截危险命令: $cmd" >&2
  exit 1   # 非 0 → Claude Code 取消本次工具调用
fi

exit 0     # 放行
```

```bash
chmod +x .claude/hooks/pre-tool-guard.sh
```

**2. 注册到 settings.json**：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": ".claude/hooks/pre-tool-guard.sh"
          }
        ]
      }
    ]
  }
}
```

**3. 验证**：让 Claude Code 执行 `rm -rf ./tmp`，会看到被拦截并附上你的提示信息，Claude 会自动换一种方式重试。

> **进阶**：把 `matcher` 换成 `"Write"` 可以拦截文件写入；换成 `"*"` 对所有工具生效。脚本里还能做日志记录、发通知、甚至交互式确认（`read -p "确认？" yn`）。

## 为什么

PostToolUse hook 是「事后补救」，PreToolUse hook 是「事前守门」。对于：
- **不可逆操作**（删文件、force push、DROP TABLE）——事后已经来不及；
- **需要审计留痕**——在工具执行前记录意图比事后更干净；
- **多人共用 AI 会话**——统一拦截规则，防止 Claude 在共享环境里越权。

有了 PreToolUse，你不再是被动接受 AI 的每一步，而是真正掌控工具调用的生杀大权。
