# 2026-06-28 | `--model` 按难度换档：复杂任务用 Opus，快速编辑用 Haiku

## 技巧

启动 Claude Code 时用 `--model` 指定模型——复杂架构设计用 Opus，日常快速改动用 Haiku，成本差距可达十倍以上，一行参数让预算花在刀刃上。

## 怎么做

**1. 启动时指定模型**

```bash
# 硬骨头：大型重构、深度 debug、复杂算法——用最强的
claude --model claude-opus-4-8

# 轻量任务：改变量名、加注释、写简单函数——用最快的
claude --model claude-haiku-4-5-20251001

# 日常均衡（系统默认）
claude --model claude-sonnet-4-6
```

**2. 把默认模型写进全局配置（一次设定，永久生效）**

```bash
# 设置全局默认模型
claude config set model claude-opus-4-8

# 查看当前默认
claude config get model

# 恢复默认 Sonnet
claude config set model claude-sonnet-4-6
```

**3. 三款模型速查**

| 模型 | ID | 适合场景 |
|---|---|---|
| Opus 4.8 | `claude-opus-4-8` | 复杂推理、大规模重构、架构设计 |
| Sonnet 4.6 | `claude-sonnet-4-6` | 日常开发（速度与能力均衡，默认） |
| Haiku 4.5 | `claude-haiku-4-5-20251001` | 快速编辑、简单查询、批量脚本 |

**4. `/fast` 切换快速输出**

在会话中输入 `/fast`，底层仍是 Opus，但以更快的速度流式输出——需要 Opus 能力又不想等的场景用它。

## 为什么

大多数人从头到尾只用默认 Sonnet，但任务对模型的需求差异很大：

- **写一个排序函数**用不着 Opus——Haiku 十秒搞定，省十倍成本
- **重构五千行核心模块**，Sonnet 可能漏掉边角 case，Opus 更稳
- **按难度换模型**，等于同样预算能跑更多高质量任务

推荐工作流：`claude config set model claude-sonnet-4-6` 设为全局默认，遇到真正的硬任务临时加 `--model claude-opus-4-8`，批量轻量操作加 `--model claude-haiku-4-5-20251001`——三档手动换，主动掌控每一分算力。
