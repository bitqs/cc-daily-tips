# 2026-06-18 | CLAUDE.md 进阶：给 Claude Code 一份「永久记忆」

## 技巧

在项目根目录放一个写得好的 `CLAUDE.md`，Claude Code 每次启动会自动读取它——相当于给 AI 一份随时生效的项目说明书，不用每次重新交代背景。

## 怎么做

**1. 创建 CLAUDE.md，写清楚四件事**

```markdown
# 项目简介
这是一个 Next.js 14 + Prisma + PostgreSQL 的 SaaS 模板。

## 常用命令
- 启动开发服务器：`npm run dev`
- 跑测试：`npm test`
- 数据库迁移：`npx prisma migrate dev`

## 代码规范
- 所有组件用函数式写法，禁止 class component
- 样式只用 Tailwind，不要内联 style
- API 路由统一放 `src/app/api/`，返回格式：`{ data, error }`

## 绝对不要做
- 不要修改 `src/lib/auth.ts`，里面有手动处理的 JWT 逻辑
- 不要删除 `prisma/seed.ts`
```

**2. 用 `@文件路径` 语法复用共享规范**

如果你有多个项目共享同一套代码规范，不需要每个项目各复制一份：

```markdown
# CLAUDE.md
@~/.claude/shared-rules/typescript-style.md
@~/.claude/shared-rules/git-conventions.md

## 项目专属配置
数据库连接在 `.env.local`，变量名 DATABASE_URL。
```

Claude Code 会自动展开 `@` 引用的文件内容。

**3. 设置全局 `~/.claude/CLAUDE.md`（跨所有项目生效）**

```bash
mkdir -p ~/.claude
cat > ~/.claude/CLAUDE.md << 'EOF'
## 我的个人偏好
- 代码注释和提交信息一律用英文
- 解释方案时先说结论，再给细节
- 遇到不确定的需求，优先问我而不是自己猜
EOF
```

这里的规则对你打开的所有项目都有效，相当于你的个人 AI 工作习惯配置文件。

**4. 检查 Claude 实际读到了什么**

在 Claude Code 里输入：

```
/memory
```

会列出当前会话加载的所有 CLAUDE.md 文件内容，方便 debug。

## 为什么

Claude Code 每次对话都是全新的——你上次告诉它"不要用 class component"，下次它又忘了。很多人花大量时间反复交代同样的背景，或者修正 Claude 不符合项目规范的输出。

`CLAUDE.md` 是解法：
- **自动加载**，不需要你每次手动粘贴上下文
- **`@import` 语法**让规范可以跨项目复用，改一处全局生效
- **全局 + 项目两级**：个人偏好放 `~/.claude/CLAUDE.md`，项目规范放项目根目录，分层管理
- 提交到 Git 后，团队所有人的 Claude Code 自动对齐同一套上下文

## 发帖草稿

每次打开 Claude Code 都要重新解释项目背景？试试在项目根目录放一个 `CLAUDE.md`——它会在每次会话自动加载，相当于给 AI 的永久记忆。

更冷门的是 `@~/.claude/shared-rules.md` 语法，可以跨多个项目复用同一套规范；再加一个全局 `~/.claude/CLAUDE.md`，把你的个人偏好（比如"先说结论""注释用英文"）永久写进去。
