# 2026-06-27 | `-c` / `-r`：关掉终端再打开，上次聊到哪接着聊

## 技巧

用 `claude -c`（continue）直接续接最近一次会话；用 `claude -r`（resume）弹出历史列表，选一条想恢复的——上下文、已读文件、任务进度全部还在。

## 怎么做

```bash
# 续接"最近一次"会话，直接继续，不开新对话
claude -c
# 或
claude --continue

# 列出历史会话，交互式选择要恢复哪一条
claude -r
# 或
claude --resume

# 非交互模式也可以指定 session-id 恢复
claude -r <session-id> -p "继续刚才的任务"
```

**配合 `-p`（非交互）使用：**

```bash
# 找到 session-id 后，在脚本里直接恢复并追加提示词
SESSION=$(claude -r --list | head -1 | awk '{print $1}')
claude --resume "$SESSION" -p "刚才改到哪了？帮我总结一下进度"
```

> 会话历史默认保存在 `~/.claude/projects/` 目录，每个项目路径对应一个子目录，文件是 `.jsonl` 格式，可以直接查看。

## 为什么

长任务最怕「上下文断裂」——写到一半被打断，下次开 Claude Code 又是一张白纸，要重新交代背景、重新贴代码、重新说需求，白费 10 分钟。

`-c` / `-r` 让会话变成「可暂停的工作流」：午饭去吃，回来 `claude -c` 继续；换台电脑（同步了 `~/.claude/` 目录），`claude -r` 找到那条会话接着干。对跑多个并行任务的开发者尤其有用——每个 worktree 配一个历史会话，随时切换，互不干扰。

上下文、任务进度、已读的文件，全都还在。
