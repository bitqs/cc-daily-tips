# 2026-06-23 用管道 + `-p` 把 Claude Code 嵌入任意 Shell 脚本

## 技巧

`claude -p` 开启非交互模式，让 Claude Code 像 `grep`/`jq` 一样接收管道输入、输出到 stdout，可直接嵌进脚本和 CI/CD。

## 怎么做

```bash
# 自动生成 commit message
git diff --staged | claude -p "用中文写一条简洁的 git commit message，遵循 Conventional Commits 格式"

# 提交前安全审查
git diff --staged | claude -p "检查这次改动有没有安全漏洞或明显 bug，有问题直接列出来"

# 分析错误日志
tail -100 app.log | claude -p "总结这些报错的根因，按严重程度排序"

# 生成 PR 描述并保存到文件
git diff main | claude -p "写一份 GitHub PR 描述，含 Summary 和 Test Plan" > pr_body.md

# 在 git pre-commit hook 里自动 review（.git/hooks/pre-commit）
#!/bin/sh
ISSUES=$(git diff --staged | claude -p "只输出代码问题，没问题就输出 OK")
if [ "$ISSUES" != "OK" ]; then
  echo "$ISSUES"
  exit 1
fi
```

`-p` / `--print`：单次执行、结果输出到 stdout、无交互界面。加 `--output-format json` 可拿到结构化输出，方便脚本解析。

```bash
git diff --staged | claude -p "列出所有函数名" --output-format json
```

## 为什么

大多数人把 Claude Code 当「对话窗口」用——手动复制代码、粘贴进去、再复制结果出来。但 `-p` 让它变成 shell 里的一等公民：任何能输出文本的命令，都能把结果喂给 Claude；任何需要文本的地方，都能接收 Claude 的输出。

这意味着：
- **git hooks 自动化**：提交前自动审查，不通过就拦截
- **CI 流水线集成**：PR 自动生成描述、changelog、release notes
- **日志分析脚本化**：`tail -f` 实时管道进来，出问题立刻告警
- **零粘贴工作流**：再也不用在终端和 Claude 网页之间反复切换
