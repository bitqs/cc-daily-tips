# 2026-06-16 | git worktree × Claude Code：同时开多个 AI 会话，互不打扰

## 技巧

用 `git worktree` 为每个任务创建独立工作目录，然后在各自目录里启动 Claude Code——多个会话同时跑，代码互不冲突。

## 怎么做

**1. 为新任务创建一个 worktree**

```bash
# 在当前仓库旁边，基于 main 新建一个分支和目录
git worktree add ../my-project-feat-login -b feat/login

# 再开一个，同时修 bug
git worktree add ../my-project-fix-crash -b fix/crash
```

执行后，`../my-project-feat-login/` 和 `../my-project-fix-crash/` 都是完整的工作目录，共享同一个 `.git`，但文件互相独立。

**2. 在每个目录里启动独立的 Claude Code 会话**

```bash
# 终端 1
cd ../my-project-feat-login
claude

# 终端 2（同时开）
cd ../my-project-fix-crash
claude
```

两个 Claude Code 实例并行运行，各自读写自己的文件，完全不会互相干扰。

**3. 任务完成后清理 worktree**

```bash
git worktree remove ../my-project-feat-login
```

**完整流程一览**

```
main repo/          ← 你平时在这里
my-project-feat-login/  ← Claude 在写登录功能
my-project-fix-crash/   ← Claude 在修崩溃 bug
```

三个目录，三个终端，一个 `.git`，零冲突。

## 为什么

Claude Code 单会话是串行的——它在写代码时你只能等。遇到"A 功能要改，B bug 也很急"的场景，很多人只能来回切换分支，或者开两个项目副本，后者还会导致 `.git` 不同步。

`git worktree` 是 Git 原生支持的多工作区方案：
- 多个目录共享同一个 `.git` 历史，`git log`、`git stash` 全部互通
- 每个目录有自己的 HEAD 和工作树，Claude Code 在里面操作绝不会踩到另一个会话
- 不需要 clone 两份仓库，磁盘占用极小
- 用完 `git worktree remove` 一行清理，干净利落

对做多功能并行开发、或者同时跑"写代码 + 写测试"两个 AI 任务的开发者来说，这是效率翻倍的冷知识。

## 发帖草稿

你有没有遇到过：Claude Code 在帮你写功能 A，但 B 的 bug 很急，却不能同时开两个任务？

试试 `git worktree`——一行命令给仓库开一个平行宇宙：

```
git worktree add ../proj-fix -b fix/crash
cd ../proj-fix && claude
```

两个目录、两个 Claude 会话同时跑，共享同一个 git 历史，文件却完全隔离。写功能、修 bug、跑测试……三件事并行，不用再等了。
