# 2026-06-15 | 自定义 Slash Command：把复杂提示词变成一个 `/命令`

## 技巧

在项目里创建 `.claude/commands/` 目录，放一个 Markdown 文件，就能在 Claude Code 里用 `/文件名` 召唤它。

## 怎么做

**1. 创建项目级命令（团队共享）**

```bash
mkdir -p .claude/commands
```

新建 `.claude/commands/review.md`，写入你的提示词模板：

```markdown
请对当前改动做 Code Review，重点检查：
1. 是否引入安全漏洞（SQL注入、XSS、命令注入）
2. 错误处理是否完整
3. 有没有不必要的副作用

用中文输出，按优先级排列问题。
```

**2. 在 Claude Code 里直接调用**

```
/review
```

Claude 会立即执行这条提示词，无需重新输入一个字。

**3. 支持 `$ARGUMENTS` 占位符（动态参数）**

`.claude/commands/commit.md`:

```markdown
为以下改动写一条 Git commit message，风格参考 Conventional Commits：

$ARGUMENTS

要求：英文，50字以内，动词开头。
```

调用时：

```
/commit 新增了用户登录的速率限制逻辑
```

**4. 个人全局命令（跨项目可用）**

```bash
mkdir -p ~/.claude/commands
# 放在这里的命令在所有项目中都能用
```

## 为什么

重度使用 Claude Code 时，有些提示词会反复输入：Code Review 模板、commit message 格式、测试用例生成规范……每次手打不仅浪费时间，还容易漏掉细节。

自定义 Slash Command 把这些"肌肉记忆"固化成命令：
- **项目级命令**提交到 Git，团队所有人共享同一套 AI 工作流
- **个人全局命令**放 `~/.claude/commands/`，跨项目复用
- 比复制粘贴提示词快 10 倍，比靠记忆描述准确 10 倍
