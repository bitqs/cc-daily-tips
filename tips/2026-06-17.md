# 2026-06-17 | PostToolUse Hook：让 Claude Code 每次改完文件自动跑 lint

## 技巧

在 `.claude/settings.json` 里配置 `PostToolUse` hook，Claude Code 每次写完文件就自动触发你的 lint / 格式化命令，输出结果直接反馈给 Claude——它会根据报错自动修正，不用你盯着。

## 怎么做

**1. 在项目根目录创建或编辑 `.claude/settings.json`**

```bash
mkdir -p .claude
```

**2. 写入 hook 配置**

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --silent 2>&1 | head -30"
          }
        ]
      }
    ]
  }
}
```

- `matcher` 支持正则，`Edit|Write` 匹配所有文件写入操作
- hook 的 stdout 会作为工具调用结果返回给 Claude，它能直接看到 lint 报错并自行修复
- `head -30` 防止输出过长撑爆上下文

**3. 常见变体**

只在改 TypeScript 文件时跑类型检查：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx tsc --noEmit 2>&1 | head -20"
          }
        ]
      }
    ]
  }
}
```

任务结束时自动跑测试（`Stop` hook）：

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "npm test --silent 2>&1 | tail -20"
          }
        ]
      }
    ]
  }
}
```

**4. 全局 hook（跨所有项目生效）**

放到 `~/.claude/settings.json`，格式相同。

## 为什么

Claude Code 改代码速度很快，但它不会主动跑 lint——改完你才发现一堆类型错误或格式问题，又要手动告诉它"帮我修一下"，来回好几轮。

`PostToolUse` hook 打通了这个循环：
- Claude 写完文件 → hook 自动触发 lint → 报错直接送回给 Claude → Claude 当场修 → 再次触发 hook 验证
- 整个「写→查→改」的闭环全自动，你只需要最后看结果
- 比手动粘贴错误信息省去大量重复对话
- 配置提交到 Git，团队所有人共享同一套质量门槛
